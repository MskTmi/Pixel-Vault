## 项目简介

Pixel Vault 是一个纯浏览器端的图片隐写工具把任意文件"塞进"一张图片里，图片照常显示、画质零损失，旁人无从察觉可选开启 AES-256-GCM 加密，即便隐藏数据被人发现，没有密码也读不出来

> 传统的 LSB 隐写会改动像素、肉眼可辨且容量有限；Pixel Vault 走的是"文件尾追加"路线——图片查看器遇到 EOF 就停了，后面的字节它根本不看

## 特性

- **任意图片 + 任意文件** —— PNG / JPG / WebP 均可，文件类型不限
- **可选 AES-256-GCM 加密** —— 密码派生用 PBKDF2-SHA256（15 万次迭代），文件名一并加密
- **画质零损失** —— 不动一个像素，原图什么样就是什么样
- **无体积上限** —— 藏多大都行，输出 = 原图 + 隐藏文件（物理守恒）
- **纯前端 · 零上传** —— 文件从不离开你的设备，无需服务器、无需账号
- **PWA 可安装** —— 加到桌面 / 主屏幕，离线也能用
- **向后兼容** —— 新版自动识别加密 / 旧版明文格式，老文件无需重新编码
- **零依赖** —— 单 HTML 文件，加密用浏览器原生 Web Crypto API

## 快速开始

### 本地运行

```bash
# 任意静态服务器即可
npx serve .
# 或
python -m http.server 8000
```

> 注意：不要直接双击 `index.html` 用 `file://` 打开——PWA / Service Worker 需要 HTTP(S) 环境，本地预览请用上面的静态服务器

### 藏文件

1. 选封面图——任意图片
2. 选秘密文件——任意类型
3. 打开加密开关（推荐）并输入密码
4. 点击 **HIDE** → 下载 `hidden_原文件名`
5. 把这张图随便发到哪都行

### 取文件

1. 切到 **REVEAL** 标签
2. 上传编码后的图片
3. 若是加密文件，输入密码
4. 点击 **Extract** → 拿回原始文件

解码器会自动嗅探格式（加密 / 旧版明文），无需手动选择

## 加密方案

| 项目       | 规格                            |
| ---------- | ------------------------------- |
| 对称算法   | AES-256-GCM（带认证标签）       |
| 密钥派生   | PBKDF2-SHA-256，150,000 次迭代  |
| Salt       | 16 字节随机数（每文件独立）     |
| IV         | 12 字节随机数（每文件独立）     |
| 完整性校验 | GCM 内置认证标签                |
| 文件名     | 与文件内容一同加密（保密）      |
| 依赖       | 无 —— 浏览器原生 Web Crypto API |

**二进制布局（加密模式）：**

```
[原图字节] [MAGIC "PVLT"] [salt 16B] [iv 12B] [密文+tag] [偏移 u32 LE] [MAGIC "PVLT"]
```

末尾 8 字节 footer 携带 payload 偏移，解码 O(1) 定位，避免在图片字节中误匹配 magic

关闭加密开关则走原始明文 `---PV-VAULT---` 标记方案，兼容历史文件

## 已知限制

- **图床剥离**：部分平台（如 Twitter）会重压缩图片并丢弃附加数据，Discord / 微信文件传输正常重要场合请先测试
- **体积守恒**：10MB 图 + 100MB 文件 = 110MB 输出，没法绕开
- **邮件拦截**：Gmail 等可能拦截此类文件，建议改用网盘链接
- **密码不可找回**：加密模式下丢失密码 = 数据永久不可恢复，**没有后门**请妥善保管

## FAQ

### 这算真正的隐写术吗？

不算严格意义的隐写（steganography）严格隐写是把数据藏进像素/频域系数里，而 Pixel Vault 走的是"文件尾追加"——图片显示无异常，但 `binwalk` / 十六进制编辑器能发现附加数据**开启加密后，发现了也读不出来**，这是它的主要防御层

如果你需要抗分析的真正隐写，请看 LSB / F5 / J-UNIWARD 等方案

### 文件会上传到服务器吗？

不会所有加解密在浏览器本地完成，不会发起任何网络请求传输你的文件唯一的外部资源是 Tailwind / 字体 CDN（已被 Service Worker 缓存）

### 能藏多大的文件？

理论上无上限，实际受浏览器内存限制百 MB 级文件实测流畅，GB 级别看设备

### 密码强度有什么要求？

工具不强制校验，但建议至少 12 位、含大小写数字符号PBKDF2 15 万次迭代能拖慢暴力破解，但弱密码依然危险推荐用密码管理器生成的随机密码

### 旧版本生成的图片还能解吗？

能解码器会自动识别加密格式（末尾 `PVLT` magic）和旧版明文格式（`---PV-VAULT---` 标记），无需手动切换
